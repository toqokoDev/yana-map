# ЖКХ Кличев (karta)

Веб-приложение для жителей г. Кличев: интерактивная карта городской инфраструктуры и сервис подачи заявок в сфере ЖКХ.

## Возможности

- **Интерактивная карта** — территории обслуживания управляющих организаций, объекты инфраструктуры (колодцы, зелёные зоны и др.), поиск по объектам.
- **Заявки** — регистрация пользователей, создание заявок с описанием, адресом, координатами и фото; редактирование и удаление своих заявок в статусах «новая» и «в работе».
- **Отображение на карте** — активные заявки показываются всегда; выполненные и отклонённые скрываются через настраиваемый срок (по умолчанию 7 и 2 дня).
- **Личный кабинет** — список своих заявок и профиль.
- **Панель администратора** — управление статусами заявок, объектами карты.
- **Руководство пользователя** — отдельная страница с инструкциями.

## Стек

| Слой | Технологии |
|------|------------|
| Backend | Node.js, Express 5, EJS |
| База данных | MySQL (сессии через `express-mysql-session`) |
| Frontend | HTML/CSS, ES-модули (без сборщика) |
| Прочее | bcryptjs, multer (загрузка фото) |

## Требования

- Node.js 18+
- MySQL 8+

## Установка и запуск

1. Клонируйте репозиторий и установите зависимости:

```bash
npm install
```

2. Скопируйте пример конфигурации и задайте переменные окружения:

```bash
cp .env.example .env
```

3. Создайте базу и таблицы, импортируйте данные карты:

```bash
npm run db:seed
```

Скрипт `db:seed` применяет `server/db/schema.sql` и загружает объекты из `scripts/objects.json`.

4. Запустите сервер:

```bash
npm start
```

Для разработки с автоперезагрузкой:

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000` (порт задаётся в `.env`).

## Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `PORT` | Порт HTTP-сервера | `3000` |
| `SESSION_SECRET` | Секрет для сессий | — |
| `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME` | Подключение к MySQL | см. `.env.example` |
| `ADMIN_LOGIN`, `ADMIN_PASSWORD` | Учётные данные администратора | — |
| `MAX_ACTIVE_REQUESTS_PER_USER` | Лимит активных заявок на пользователя | `5` |
| `MAP_VISIBLE_DAYS_DONE` | Дней показа выполненной заявки на карте | `7` |
| `MAP_VISIBLE_DAYS_REJECTED` | Дней показа отклонённой заявки на карте | `2` |

## Структура проекта

```
karta/
├── server/           # Express-приложение (маршруты, API, БД)
├── views/            # EJS-шаблоны страниц
├── js/               # Клиентские модули (карта, заявки, API)
├── styles/           # Стили
├── images/           # Статика (иконки, иллюстрации)
├── uploads/          # Загруженные фото заявок
├── scripts/          # Импорт данных карты (db:seed)
├── docs/             # Документация API
└── server.js         # Точка входа (делегирует в server/index.js)
```

## API

REST API доступен по префиксу `/api`. Полное описание эндпоинтов, форматов запросов и ответов — в [docs/API.md](docs/API.md).

Краткий справочник в JSON: `GET /api`.
